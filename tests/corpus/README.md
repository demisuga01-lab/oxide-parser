# Poppler Parity Corpus

This directory contains the tagged PDF corpus used by `scripts/poppler_compare.py`.

The corpus is generated by:

```powershell
py scripts\generate_parity_corpus.py --allow-missing
```

The generated `manifest.json` maps each PDF to a primary category, source, license,
SHA-256, and notes. The harness reads that manifest so reports can aggregate by
category.

## Sources

- `pdfs/existing/`: copies of existing engine fixtures from
  `crates/engine/tests/fixtures`.
- `pdfs/pdfjs/`: selected fixtures from Mozilla PDF.js `test/pdfs`, licensed
  under Apache-2.0 by the upstream project.
- `pdfs/generated/`: small CC0 synthetic PDFs generated by
  `scripts/generate_parity_corpus.py` for reproducible text, image-only,
  vector, form, and 120-page performance coverage.

## Categories

- `scanned`
- `jpeg2000`
- `encrypted`
- `complex-vector`
- `forms`
- `cjk-text`
- `rtl-text`
- `multi-column`
- `large-multipage`
- `text-basic`

## Running The Harness

Poppler is a dev dependency for the harness only. It is not an Oxide runtime
dependency.

```powershell
py scripts\poppler_compare.py `
  --manifest tests\corpus\manifest.json `
  --poppler-bin-dir target\tools\poppler\poppler-26.02.0\Library\bin `
  --output-dir target\poppler_compare\baseline `
  --report-path docs\poppler_parity_baseline.md
```

If Poppler is on `PATH`, omit `--poppler-bin-dir`.

Useful focused runs:

```powershell
py scripts\poppler_compare.py --category cjk-text
py scripts\poppler_compare.py --file vertical.pdf
py scripts\poppler_compare.py --max-render-pages 0
```

`--max-render-pages 0` compares all rendered pages. The default is one rendered
page per PDF so the full corpus is fast enough to run frequently.
