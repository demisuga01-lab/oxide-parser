# PDF/A and PDF/UA Compliance

Oxide includes a scoped compliance layer for archival and accessibility
workflows.

## PDF/A Validation

Use `validate_pdfa` with `PdfAProfile::PdfA1B` or `PdfAProfile::PdfA2B`.
The validator returns a structured report with rule ids, locations, severity,
and messages.

Implemented checks include:

- no encryption
- catalog XMP metadata with `pdfaid:part` and `pdfaid:conformance`
- XMP synchronization for present `/Info` fields such as title, author,
  subject, keywords, creator, and producer
- OutputIntent with an ICC profile stream
- all fonts embedded
- no JavaScript, launch actions, automatic actions, or embedded files for the
  supported PDF/A-1/2 profiles
- PDF/A-1 transparency rejection

```rust
use oxide_engine::{validate_pdfa, PdfAProfile, PdfDocument};

# fn main() -> oxide_engine::Result<()> {
let doc = PdfDocument::open_path("input.pdf")?;
let report = validate_pdfa(&doc, PdfAProfile::PdfA2B)?;
for violation in &report.violations {
    eprintln!("{}: {}", violation.rule, violation.message);
}
# Ok(())
# }
```

## PDF/A Conversion

`convert_to_pdfa_checked` converts eligible PDFs to PDF/A-2b or PDF/A-1b and
then revalidates the result with Oxide.

The converter is intentionally conservative. It adds conforming XMP metadata
synced with the retained `/Info` fields, adds an sRGB OutputIntent ICC profile
stream generated by Oxide, strips disallowed JavaScript/launch/automatic action
entries, and writes through the profile-appropriate writer: classic PDF 1.4
output for PDF/A-1b, and the modern xref-stream/object-stream writer for
PDF/A-2b. It refuses to convert documents with non-embedded fonts, because the
original font programs are not available inside those PDFs. That is reported as
a conversion blocker rather than silently substituting fonts.

```rust
use oxide_engine::{convert_to_pdfa_checked, PdfAProfile, PdfDocument};

# fn main() -> oxide_engine::Result<()> {
let doc = PdfDocument::open_path("embedded-font-input.pdf")?;
let (bytes, report) = convert_to_pdfa_checked(&doc, PdfAProfile::PdfA2B)?;
assert!(report.validation.compliant);
std::fs::write("archival.pdf", bytes)?;
# Ok(())
# }
```

## PDF/UA

`validate_pdfua` checks the basic accessibility requirements that can be
verified mechanically: document language, marked/tagged status, a structure tree,
and alternate text on Figure structure elements.

`improve_pdfua_best_effort` can add `/Lang`, `/MarkInfo`, and a minimal
`/StructTreeRoot` when missing. This is assistive tagging only, not a guarantee
of full PDF/UA conformance. True PDF/UA usually requires human review of reading
order, headings, figure alternate text, and table semantics.

## veraPDF

veraPDF is the external reference validator for PDF/A and PDF/UA. The Oxide
validator is useful for deterministic SDK checks and conversion gates. When
veraPDF is available in the environment, use it as the final oracle for
regulatory certification.

## Example

```bash
cargo run -p oxide-engine --example compliance -- target/compliance-demo
qpdf --check target/compliance-demo/compliance-pdfa-2b.pdf
```

The example writes an embedded-font source PDF, converted PDF/A-2b and PDF/A-1b
PDFs, and a best-effort PDF/UA-improved PDF.
