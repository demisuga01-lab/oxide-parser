# PDF/A and PDF/UA Compliance

Oxide includes a scoped compliance layer for archival and accessibility
workflows.

## PDF/A Validation

Use `validate_pdfa` with `PdfAProfile::PdfA1B`, `PdfA2B`, `PdfA2A`,
`PdfA3B`, or `PdfA3A`. The validator returns a structured report with rule
ids, locations, severity, and messages.

Implemented checks include:

- no encryption
- catalog XMP metadata with `pdfaid:part` and `pdfaid:conformance`
- XMP synchronization for present `/Info` fields such as title, author,
  subject, keywords, creator, and producer
- OutputIntent with an ICC profile stream
- all fonts embedded
- no JavaScript, launch actions, or automatic actions
- no embedded files for PDF/A-1 and PDF/A-2
- PDF/A-3 embedded files must be represented by a FileSpec with
  `/AFRelationship`
- PDF/A Level A profiles require `/Lang`, `/MarkInfo`, a resolvable
  non-empty `/StructTreeRoot`, standard structure roles, and alternate text on
  Figure structure elements
- PDF/A-1 transparency rejection

```rust
use oxide_engine::{validate_pdfa, PdfAProfile, PdfDocument};

# fn main() -> oxide_engine::Result<()> {
let doc = PdfDocument::open_path("input.pdf")?;
let report = validate_pdfa(&doc, PdfAProfile::PdfA2B)?;
for violation in &report.violations {
    eprintln!("{}: {}", violation.rule, violation.message);
}
# Ok(())
# }
```

## PDF/A Conversion

`convert_to_pdfa_checked` converts eligible PDFs to PDF/A-1b, PDF/A-2b,
PDF/A-2a, PDF/A-3b, or PDF/A-3a and then revalidates the result with Oxide.

The converter is intentionally conservative. It adds conforming XMP metadata
synced with the retained `/Info` fields, adds an sRGB OutputIntent ICC profile
stream generated by Oxide, strips disallowed JavaScript/launch/automatic action
entries, and writes through the profile-appropriate writer: classic PDF 1.4
output for PDF/A-1b, and the modern xref-stream/object-stream writer for
PDF/A-2 and PDF/A-3. For Level A conversion it adds deterministic tagging
scaffolding (`/Lang`, `/MarkInfo`, and a non-empty structure root) when the
source is untagged. For PDF/A-3 it preserves embedded files and repairs
FileSpec dictionaries missing `/AFRelationship` by assigning `/Unspecified`.
It refuses to convert documents with non-embedded fonts, because the original
font programs are not available inside those PDFs. That is reported as a
conversion blocker rather than silently substituting fonts.

```rust
use oxide_engine::{convert_to_pdfa_checked, PdfAProfile, PdfDocument};

# fn main() -> oxide_engine::Result<()> {
let doc = PdfDocument::open_path("embedded-font-input.pdf")?;
let (bytes, report) = convert_to_pdfa_checked(&doc, PdfAProfile::PdfA2B)?;
assert!(report.validation.compliant);
std::fs::write("archival.pdf", bytes)?;
# Ok(())
# }
```

## PDF/UA

`validate_pdfua` checks the accessibility requirements Oxide can verify
mechanically: document language, marked/tagged status, a resolvable non-empty
structure tree, standard structure roles, and alternate text on Figure
structure elements.

`improve_pdfua_best_effort` can add `/Lang`, `/MarkInfo`, and a minimal
non-empty `/StructTreeRoot` when missing. This is assistive tagging only, not a
guarantee of full PDF/UA conformance. veraPDF UA-1 still requires tagged or
artifact-marked content, PDF/UA identification metadata, viewer preferences,
and human-reviewed reading order, headings, figure alternate text, and table
semantics.

## veraPDF

veraPDF is the external reference validator for PDF/A and PDF/UA. GA Prompt 3
validated the generated matrix with bundled veraPDF 1.30.2:

| Profile | Generated example | veraPDF result |
| --- | --- | --- |
| PDF/A-1b | `compliance-pdfa-1b.pdf` | PASS |
| PDF/A-2b | `compliance-pdfa-2b.pdf` | PASS |
| PDF/A-2a | `compliance-pdfa-2a.pdf` | PASS |
| PDF/A-3b | `compliance-pdfa-3b.pdf` | PASS |
| PDF/A-3a | `compliance-pdfa-3a.pdf` | PASS |

The same smoke file improved with `improve_pdfua_best_effort` is intentionally
not claimed as veraPDF PDF/UA-1 compliant; veraPDF reports remaining semantic
tagging and PDF/UA metadata requirements. Use veraPDF as the final oracle for
regulatory certification.

## Example

```bash
cargo run -p oxide-engine --example compliance -- target/compliance-demo
qpdf --check target/compliance-demo/compliance-pdfa-2b.pdf
```

The example writes an embedded-font source PDF, converted PDF/A-1b, PDF/A-2b,
PDF/A-2a, PDF/A-3b, PDF/A-3a PDFs, and a best-effort PDF/UA-improved PDF.
