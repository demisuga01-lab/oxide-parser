# Embedded File Attachments (`detach`)

`oxide detach` lists and extracts **embedded files** (attachments) from a PDF —
the `pdfdetach`-equivalent. PDFs can carry arbitrary embedded files:
spreadsheets, source data, other PDFs, and XML e-invoices (ZUGFeRD / Factur-X
embed an XML invoice inside the PDF).

```
oxide detach in.pdf --list                       # list attachments
oxide detach in.pdf --list --json                # machine-readable listing
oxide detach in.pdf --save 1   --output-dir out  # extract by index
oxide detach in.pdf --name data.xml --output-dir out   # extract by name
oxide detach in.pdf --save-all --output-dir out  # extract all
oxide detach secret.pdf --save-all --password p  # encrypted input
```

## Where embedded files are located

The engine (`oxide_engine::attachments`) collects embedded files from **both**
standard locations and dedupes by the embedded-file stream's object id (a file
referenced from both places is listed once):

1. **Catalog `/Names /EmbeddedFiles` name tree** — the primary location. A
   generic, cycle-safe, depth-bounded name-tree walker (`walk_name_tree`)
   traverses the balanced `/Kids`/`/Names` tree, yielding `(name, filespec)`
   pairs. The walker is reusable for any PDF name tree (Dests, JavaScript, …).
2. **Page `/Annots` of `/Subtype /FileAttachment`** — "paperclip" attachments
   placed on a page; their `/FS` is a file specification resolved the same way.

For each file specification (`/Type /Filespec`):

- **name** — `/UF` (Unicode) preferred, else `/F`, decoded from PDF text-string
  form (UTF-16BE-with-BOM or PDFDocEncoding).
- **description** — `/Desc`.
- **embedded-file stream** — `/EF /F` (fallback `/EF /UF`), a normal PDF stream
  (often FlateDecode-compressed) whose decoded bytes are the file content.
- **metadata** from the stream's `/Params`: `/Size`, `/CreationDate`,
  `/ModDate`, and `/CheckSum` (MD5 of the uncompressed data).

### Listing fields

`Attachment { index, name, description, size, creation_date, mod_date,
checksum_md5, stream_object, stream_generation, source }` — `source` is
`name_tree` or `annotation`. `--json` serializes this directly.

## Extraction

Extraction fetches the embedded-file stream and decodes it through the existing
filter pipeline (`decode_stream`), recovering the exact file bytes. When the
filespec carries a `/Params /CheckSum`, the decoded bytes are MD5-checked
against it and a **mismatch is logged as a warning** (not a hard error — some
producers store an absent or wrong checksum).

### Filename sanitization (security)

The embedded file **name is attacker-controlled** and may contain path
traversal (`../`), absolute paths, drive letters, or separators. A naive
extractor that joins the raw name onto an output directory is a path-traversal
vulnerability. `sanitize_filename` defends against this:

- reduces the name to its **final path component** (splitting on both `/` and
  `\`),
- strips control characters and any remaining separator/colon/NUL,
- rejects pure-dot names (`.`, `..`) and empty results, falling back to
  `attachment.bin`.

So `../../etc/passwd` → `passwd`, `C:\Users\x\evil.dll` → `evil.dll`, and the
written file can never escape the chosen `--output-dir`. This is verified by a
test that extracts an attachment literally named `../../evil.txt` and confirms
the byte payload lands as `<output-dir>/evil.txt` with nothing escaping.

## Validation

`crates/engine/tests/attachments.rs` (fixtures generated by
`scripts/make_attachment_fixtures.py`):

- **List**: correct count, names, sizes, description, and stored MD5 checksum
  for a name-tree fixture; the annotation fixture's file is found.
- **Extract**: byte-exact equality with the known original payloads
  (name-tree text and a 512-byte binary annotation payload).
- **Security**: a `../../evil.txt` attachment sanitizes to `evil.txt` and stays
  inside the output directory.
- **Empty**: a PDF with no attachments lists empty (not an error).
- **Dedupe**: no duplicate stream ids in a listing.
- **`pdfdetach` cross-check**: attachment counts agree across all fixtures, and
  Oxide-extracted bytes equal `pdfdetach -saveall` bytes for the name-tree file.
